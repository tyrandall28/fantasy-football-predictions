# Fantasy Football Predictor V2.0 - Development Roadmap

This document outlines the planned improvements for Version 2.0 of the fantasy football prediction model. Each section includes specific implementation steps and expected outcomes to enhance model performance and interview presentation value.

## 1. Model Comparison & Selection

### Objective
Demonstrate understanding that model selection is crucial by comparing multiple algorithms and selecting the best performer.

### Implementation Steps
- [ ] **Add Random Forest Regressor**
  - Implement with similar hyperparameters to current GBR
  - Good for handling non-linear relationships and feature interactions
  
- [ ] **Add XGBoost Regressor** 
  - Install xgboost: `pip install xgboost`
  - Known for superior performance on tabular data competitions
  - Better handling of missing values and regularization
  
- [ ] **Add Linear Regression Baseline**
  - Simple baseline to show improvement from ensemble methods
  - Useful for understanding feature relationships
  
- [ ] **Create Model Comparison Script**
  - Train all models with same train/test split
  - Compare MAE, R², and Spearman correlation
  - Generate comparison table for documentation

### Expected Outcome
Show interviewers that I systematically evaluate multiple approaches rather than defaulting to one model.

## 2. Hyperparameter Optimization

### Objective
Demonstrate knowledge of proper model tuning methodologies and ability to systematically improve performance.

### Implementation Steps
- [ ] **Implement GridSearchCV for GradientBoostingRegressor**
  - Parameters to tune: `n_estimators`, `max_depth`, `learning_rate`, `subsample`
  - Use 3-fold cross-validation to avoid overfitting
  
- [ ] **Create parameter grid**:
  ```python
  param_grid = {
      'n_estimators': [300, 500, 700],
      'max_depth': [3, 5, 7],
      'learning_rate': [0.05, 0.1, 0.15],
      'subsample': [0.8, 0.9, 1.0]
  }
  ```

- [ ] **Document performance improvements**
  - Before/after tuning comparison
  - Show optimal parameters found
  - Quantify performance gain (even if small)

- [ ] **Add RandomizedSearchCV option**
  - For larger parameter spaces when GridSearch is too expensive
  - Shows understanding of computational trade-offs

### Expected Outcome
Demonstrate systematic approach to model optimization and understanding of hyperparameter impact.

## 5. Advanced Feature Engineering

### Objective
Show creativity and domain expertise in creating predictive features.

### Implementation Steps
- [ ] **Market Share Metrics**
  - Calculate target share: `player_targets / team_total_targets`
  - Calculate touch share: `(rushes + targets) / team_total_touches`
  - Red zone usage metrics
  
- [ ] **Team Context Features**
  - Team pass/run ratio from previous season
  - Offensive line rankings (if data available)
  - Coaching changes (binary feature)
  
- [ ] **Advanced Efficiency Metrics**
  - Yards per target, yards per touch
  - Target depth (air yards if available)
  - Situational stats (3rd down conversion rate, red zone efficiency)
  
- [ ] **Strength of Schedule**
  - Average points allowed by upcoming opponents
  - Defensive rankings against position

### Expected Outcome
Demonstrate advanced feature engineering skills and deep sports analytics knowledge.
